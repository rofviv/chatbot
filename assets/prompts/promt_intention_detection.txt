CREATE_ORDER
Este flujo debe activarse cuando el usuario quiera crear un pedido o ver el menu. Detecta intenciones relacionadas con la solicitud de crear un pedido o ver el menu.

Ejemplo de intenciones:
- Quiero crear un pedido
- Quiero hacer un pedido
- Quiero ordenar
- Quiero comprar
- Menu
- Cual es el menu?
- Ver el menu
- Quiero ver el menu

STATUS_ORDER
Este flujo debe activarse cuando el usuario quiera consultar el estado de su pedido. Detecta intenciones relacionadas con la solicitud de consultar el estado de un pedido.

Ejemplo de intenciones:
- Quiero saber el estado de mi pedido
- Quiero ver el estado de mi pedido
- Quiero consultar el estado de mi pedido
Criterio:
El usuario esta interesado en consultar el estado de su pedido.

CANCEL_ORDER
Este flujo debe activarse cuando el usuario quiera cancelar su pedido. Detecta intenciones relacionadas con la solicitud de cancelar un pedido.

Ejemplo de intenciones:
- Quiero cancelar mi pedido
- Cancelar mi pedido
- Ya no quiero mi pedido
Criterio:
El usuario esta interesado en cancelar su pedido.

QUESTION_LOCATION
Este flujo debe activarse cuando el usuario quiera saber porque se le esta pidiendo su ubicación. Detecta intenciones relacionadas con la solicitud de saber porque se le esta pidiendo su ubicación.

Ejemplo de intenciones:
- Porque me estas pidiendo mi ubicación?
- No entiendo porque me estas pidiendo mi ubicación
- Porque necesitas mi ubicación?
- Que necesito para hacer un pedido?

Criterio:
El usuario esta interesado en saber porque se le esta pidiendo su ubicación.
